# PID控制仿真平台的设计与实现

## 1 引言

PID控制器自1922年被Nicholas Minorsky提出以来，至今仍是工业过程控制中应用最广泛的控制算法。据估计，工业现场90%以上的控制回路采用PID或其变体。然而，PID参数整定长期以来依赖工程师的经验，教科书上的整定公式在面对实际系统时往往需要反复试凑。

本文介绍一个PID控制仿真与调试平台（pid-lab）的设计与实现。该平台集成了多种PID算法、被控对象模型、自动整定方法和频域分析工具，可用于控制工程课程教学、PID参数整定实验以及控制系统设计的前期验证。

## 2 系统架构

平台采用前后端分离架构。后端基于FastAPI框架，通过WebSocket实现仿真数据的实时推送；前端采用原生JavaScript和Canvas绘制示波器风格的波形显示。

```
┌─────────────┐    WebSocket     ┌─────────────┐
│   Browser   │ ◄──────────────► │   Server    │
│  (Canvas)   │    JSON frames   │  (FastAPI)  │
└─────────────┘                  └──────┬──────┘
                                        │
                                 ┌──────┴──────┐
                                 │  Simulation │
                                 │   Engine    │
                                 └─────────────┘
```

仿真引擎以固定步长（默认0.1秒）推进，在每个采样时刻完成以下步骤：

1. 读取当前过程变量PV
2. 计算误差e = SP - PV
3. 通过PID算法计算控制量CV
4. 将CV送入被控对象模型，得到下一步的PV
5. 将SP、PV、CV和性能指标打包推送给前端

这种离散化的仿真方式与实际数字控制系统的采样机制一致，仿真结果可以直接指导实际参数整定。

## 3 PID算法实现

### 3.1 三种标准形式

平台实现了三种常用的PID算法形式。

**并行形式（Parallel Form）**

```
u(t) = Kp·e(t) + Ki·∫e(τ)dτ + Kd·de(t)/dt
```

这是教科书中最常见的形式，三个增益Kp、Ki、Kd相互独立，物理意义直观。

**ISA标准形式**

```
u(t) = Kp·[e(t) + (1/Ti)·∫e(τ)dτ + Td·de(t)/dt]
```

ISA形式将积分和微分作用表示为时间常数Ti（积分时间）和Td（微分时间）。这种形式在过程控制领域广泛使用，其优点是Ti和时间常数有直接的物理对应关系。

**增量式（Incremental Form）**

```
Δu(k) = Kp·[e(k)-e(k-1)] + Ki·e(k)·Δt + Kd·[e(k)-2e(k-1)+e(k-2)]/Δt
```

增量式算法计算的是控制量的变化量而非绝对值。其天然具有抗积分饱和特性——当执行器饱和时，增量项自动被截断，不会产生积分累积。

### 3.2 工程实现细节

**积分抗饱和**

积分饱和（Integral Windup）是PID控制中最常见的工程问题之一。当执行器达到物理极限（如阀门全开）时，误差持续存在导致积分项不断累积，使得系统在误差反向后需要很长时间才能退出饱和状态。

平台实现了两种抗饱和策略：

- 限幅法（Clamping）：检测到输出饱和且误差方向与饱和方向一致时，冻结积分项。
- 反馈抑制法（Back-calculation）：将实际输出与理想输出的差值乘以一个增益后反馈到积分项，使积分项自动退饱和。

**微分滤波**

理想微分项对高频噪声极为敏感。在实际系统中，测量噪声经过微分后会被放大，导致控制量剧烈波动。平台在微分项上施加一阶低通滤波：

```
D_filtered = α·D_filtered_prev + (1-α)·D_raw
α = Tf / (Tf + Δt)
```

其中Tf为滤波时间常数，通常取采样时间的5~10倍。

**设定值加权**

直接对设定值做微分会产生阶跃响应时的微分冲击。平台支持设定值加权（Setpoint Weighting），通过参数β和γ分别调节比例项和微分项对设定值的权重：

```
e_P = β·SP - PV    (比例项误差)
e_D = γ·SP - PV    (微分项误差)
```

当β<1或γ<1时，相当于"比例微分先行"，可以有效减小设定值阶跃时的超调。

**无扰切换**

在工程实践中，经常需要在运行中切换PID算法或修改参数。如果直接切换，由于积分项和微分项的状态不连续，会导致输出跳变。平台在参数切换时通过反算积分项的初始值，使新旧算法在切换点的输出保持一致。

## 4 被控对象模型

### 4.1 内置模型

平台提供了8种典型的被控对象模型，覆盖了过程控制、运动控制和电力电子等领域的常见场景。

| 模型 | 传递函数 | 典型应用 |
|------|----------|----------|
| 电机调速 | K/(τs+1) | 直流电机、温度回路 |
| 温度控制 | Ke^(-θs)/(Ts+1) | 加热炉、化工反应器 |
| 液位控制 | K/(s(Ts+1)) | 储罐液位、积分过程 |
| 伺服定位 | K/(s(τ₁²s²+2ζ₁τ₁s+1)(τ₂s+1)) | 机械臂、数控机床 |
| 二阶系统 | ωn²/(s²+2ζωns+ωn²) | 弹簧-质量-阻尼系统 |
| 耦合双容 | G₁(s)+G₂(s) | 双容水箱、耦合回路 |
| 二阶加滞后 | Ke^(-θs)/((τ₁s+1)(τ₂s+1)) | 工业加热过程 |
| 纯积分 | K/s | 液位、角度积分 |

### 4.2 纯滞后实现

纯滞后（Dead Time）是过程控制中最难处理的环节之一。传统的Pade近似虽然可以用有理传递函数逼近延迟，但会引入右半平面零点，导致仿真不稳定。

平台采用延迟线（Delay Line）实现纯滞后：维护一个FIFO缓冲区，长度等于延迟时间除以采样时间。每个采样时刻，将模型输出写入缓冲区尾部，从头部读出作为实际输出。这种方法精确且数值稳定。

对于频域分析（Bode图、Nyquist图），由于需要有理传递函数形式，仍然使用三阶Pade近似来表示延迟环节。

### 4.3 自定义传递函数

用户可以直接输入分子和分母多项式的系数，实现任意传递函数的仿真。系数按降幂排列，用逗号分隔。例如，输入分子"1,0"和分母"1,2,1"表示传递函数：

```
G(s) = s / (s² + 2s + 1)
```

这一功能使得平台可以用于用户自己的被控对象，不限于内置模型。

## 5 自动整定

### 5.1 经典整定方法

**Ziegler-Nichols阶跃响应法**

对开环系统施加阶跃输入，记录响应曲线。通过切线法辨识三个参数：静态增益K、等效延迟时间L和等效时间常数T。然后按以下公式计算PID参数：

```
Kp = 1.2T/(KL),  Ti = 2L,  Td = 0.5L
```

**Cohen-Coon方法**

使用相同的辨识过程，但采用不同的整定公式，在归一化延迟较小的情况下比Z-N方法有更好的动态性能。

**继电反馈法**

用一个继电器（Bang-Bang控制器）代替PID控制器，使系统产生等幅振荡。从振荡中测量临界增益Ku和临界周期Tu，然后应用Z-N整定公式。这种方法不需要开环试验，可以在闭环状态下自动完成。

**Lambda整定法**

用户指定期望的闭环时间常数λ，算法根据辨识出的过程模型计算PI参数。Lambda整定法的特点是可以通过调节λ来权衡响应速度和鲁棒性——λ越大，响应越慢但对模型不确定性越不敏感。

### 5.2 参数优化

除了基于公式的整定方法，平台还支持基于数值优化的参数寻优。用户选择优化目标（ISE、IAE、ITAE或超调量），算法使用Nelder-Mead单纯形法在参数空间中搜索最优解。

优化过程如下：

1. 用Z-N方法获得初始猜测
2. 以初始猜测为起点，构造目标函数J(Kp, Ki, Kd)
3. 目函数通过完整闭环仿真计算：给定PID参数→仿真→计算性能指标
4. Nelder-Mead迭代搜索，直到收敛

ISE（积分平方误差）对大误差敏感，适合需要快速消除大偏差的场合；IAE（积分绝对误差）对大小误差等权；ITAE（时间加权绝对误差）对长时间存在的误差施加更大惩罚，有助于减少振荡。

## 6 频域分析

### 6.1 Bode图

平台计算开环传递函数L(s) = C(s)·G(s)的频率响应，绘制幅频特性和相频特性曲线。同时自动计算两个关键稳定性指标：

- 增益裕度（Gain Margin）：相位穿越-180°时，幅值距离0dB的余量
- 相位裕度（Phase Margin）：增益穿越0dB时，相位距离-180°的余量

一般工程经验要求相位裕度在30°~60°之间，增益裕度大于6dB。

### 6.2 Nyquist图

Nyquist图以极坐标形式展示开环频率响应，正频率和负频率分别绘制。图中标注(-1, j0)点——根据Nyquist稳定判据，开环频率响应曲线包围(-1, j0)点的次数决定了闭环系统的稳定性。

### 6.3 含延迟系统的处理

对于含纯滞后的系统，频域分析使用Pade近似将延迟环节转化为有理传递函数。三阶Pade近似在频率低于1/(2θ)时具有足够的精度，可以满足工程分析的需要。

## 7 系统辨识

### 7.1 一阶加滞后辨识

平台支持从实测阶跃响应数据辨识一阶加滞后（FOPTD）模型。辨识步骤如下：

1. 从响应曲线读取稳态值，计算静态增益K
2. 对响应曲线做数值微分，找到斜率最大的点（拐点）
3. 过拐点做切线，切线与零线的交点给出等效延迟L，切线斜率给出等效时间常数T

同时计算R²拟合度，评估辨识质量。

### 7.2 ARX模型辨识

对于更一般的系统，平台支持ARX（自回归外生）模型辨识。给定输入u和输出y的观测数据，构造回归矩阵：

```
y(k) = -a₁y(k-1) - ... - any(k-n) + b₀u(k-1) + ... + bₙu(k-n)
```

通过最小二乘法求解参数向量[a₁,...,aₙ, b₀,...,bₙ]，得到离散传递函数的分子和分母系数。

## 8 高级控制结构

### 8.1 串级控制

串级控制由外环（主控制器）和内环（副控制器）组成。外环的输出作为内环的设定值。内环通常采用高增益快速响应，抑制进入内环的扰动；外环负责跟踪主设定值，消除稳态误差。

### 8.2 前馈控制

当扰动可以测量时，可以在反馈回路之外增加前馈通道，根据扰动的测量值直接计算补偿量。理想前馈可以完全消除可测扰动的影响，但需要准确知道扰动到输出的传递函数。

### 8.3 Smith预估器

对于大滞后系统，Smith预估器通过引入过程模型来"抵消"滞后环节的影响。控制器看到的反馈信号是经过滞后补偿后的等效输出，从而可以像控制无滞后系统一样进行参数整定。

## 9 使用方法

### 9.1 环境准备

```bash
git clone <repository-url>
cd pid-lab
pip install fastapi uvicorn scipy numpy
python main.py
```

浏览器访问 http://localhost:8000。

### 9.2 基本操作

1. 选择被控对象模型（如"温度控制"）
2. 点击RUN开始仿真
3. 拖动Kp、Ki、Kd滑条，观察波形变化
4. 点击"整定"按钮，选择自动整定方法
5. 点击"频域"按钮，查看Bode图和Nyquist图

### 9.3 从实测数据辨识

1. 准备CSV文件，第一列为时间，第二列为输入，第三列为输出
2. 点击"导入"按钮，选择文件
3. 系统自动辨识K、L、T参数并显示拟合度

### 9.4 参数优化

1. 点击"整定"按钮
2. 选择"参数优化"方法
3. 选择优化目标（ISE/IAE/ITAE）
4. 点击"执行"，等待优化结果
5. 点击"应用参数"将最优参数填入

### 9.5 自定义传递函数

1. 在模型下拉菜单中选择"自定义TF"
2. 输入分子系数（如"1"）和分母系数（如"1,2,1"）
3. 系数按s的降幂排列，逗号分隔

## 11 结语

pid-lab平台将PID控制的理论分析、仿真验证和参数整定集成在一个轻量级的Web应用中。通过示波器风格的实时波形显示和丰富的分析工具，使用者可以直观地理解PID参数对系统性能的影响，快速完成参数整定。

平台的代码完全开源，不依赖商业软件，可以在任何安装了Python的计算机上运行。对于控制工程的学习者和研究者，这是一个可以随身携带的"虚拟实验室"。
